Вот готовый боевой модуль md_cleaner.py, оптимизированный для интеграции в вашу консольную утилиту автоматизации (CLI):pythonimport re
from pathlib import Path

def clean_markdown_spaces(file_path: str | Path) -> bool:
    """
    Проверяет и исправляет дублирующиеся пустые строки.
    Возвращает True, если файл был изменен, иначе False.
    """
    path = Path(file_path)
    if not path.exists():
        return False
        
    content = path.read_text(encoding='utf-8')
    
    # 1. Нормализуем строки с пробелами (превращаем строки, где только пробелы/табы, в чисто пустые)
    # Это предотвращает обход регулярного выражения строками вида '\n   \n\n'
    normalized = re.sub(r'\n[ \t]+\n', '\n\n', content)
    
    # 2. Схлопываем 3 и более переносов строк в 2 (то есть оставляем ровно одну пустую строку)
    cleaned = re.sub(r'\n{3,}', '\n\n', normalized)
    
    # Если изменений нет, не трогаем файл (важно для сохранения mtime/атрибутов файла)
    if content == cleaned:
        return False
        
    path.write_text(cleaned, encoding='utf-8')
    return True
Используйте код с осторожностью.Как это интегрировать в ваши режимы:1. Быстрый режим (--fast / только измененные)В этом режиме ваша автоматизация (например, через git diff --name-only или стейдж-менеджер) получает список только что измененных агентом файлов. Вы прогоняете их через функцию очистки до того, как сработают ваши скрипты автоиндексации.Плюс: Исправление происходит мгновенно (доли миллисекунды на файл), индексы всегда получают чистый Markdown.2. Полный режим (--full / по всей базе)Запускается ретроспективно или по расписанию. Проходит циклом по всем .md файлам в репозитории/базе.Плюс: Исправляет старые файлы, которые могли быть изменены вручную или старыми версиями промптов ИИ-агента. Благодаря проверке if content == cleaned:, скрипт не будет перезаписывать файлы, которые уже в порядке, что сбережет ресурсы диска.💡 Совет по расширению линтераРаз вы уже пишете автоматический линтер для вывода ИИ, добавьте туда еще одно регулярное выражение, которое часто ломает скрипты индексации после работы LLM — пробелы на концах строк заголовков.ИИ часто пишет ## Заголовок , что приводит к тому, что парсер считывает имя заголовка вместе с пробелом на конце. Избавиться от этого можно одной строкой:python# Удаляет пробелы на концах строк, начинающихся с #
cleaned = re.sub(r'^(#+ .*?)[ \t]+$', r'\1', cleaned, flags=re.MULTILINE)
