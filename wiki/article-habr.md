В нашем случае мы выбрали почасовую прерванную GPU от <a href="https://selectel.ru/services/cloud/servers/gpu/" rel="noopener noreferrer nofollow">Selectel</a>.</p><p><strong>Характеристики виртуальной машины:</strong></p><ul><li><p>8 vCPU, 32 ГБ RAM, 200 ГБ SSD</p></li><li><p>1 × RTX 4090 (24 ГБ VRAM)</p></li><li><p>Ubuntu 24.04 LTS 64-bit, GPU Driver 580 (Open)</p></li><li><p>Локация: Москва / ru-7b</p></li></ul><p>После запуска сервера и перехода в терминал выполняем следующие команды:</p><h3>Шаг 1. Сборка llama-server, llama-bench и llama-cli</h3><p>Каждый инструмент решает задачу, поднять REST API (<code>llama-server</code>), провести тестирование оптимальных параметров размера контекста вход и выход (llama-bench).</p><p>Команды выполняются в терминале: </p><p><strong>1. Скачать проект</strong></p><pre><code class="bash">git clone https://github.com/ggml-org/llama.cpp
cd llama.cpp
mkdir build &amp;&amp; cd build</code><div class="code-explainer"><a href="https://sourcecraft.dev/" class="tm-button code-explainer__link" style="visibility: hidden;"><img style="width:87px;height:14px;object-fit:cover;object-position:left;"/></a></div></pre><p><strong>2. Собрать исполняемые файлы</strong></p><pre><code class="bash"># Указываем путь к CUDA-компилятору
export CUDACXX=/usr/local/cuda/bin/nvcc

cmake .. \
  -DCMAKE_BUILD_TYPE=Release \
  -DGGML_CUDA=ON \
  -DLLAMA_CURL=ON \
  -DGGML_CUDA_FA=ON \          # Flash Attention для ускорения
  -DCMAKE_CUDA_ARCHITECTURES=89 # Архитектура для RTX 40xx

cmake --build . --config Release \
  --target llama-server llama-bench llama-cli --parallel</code><div class="code-explainer"><a href="https://sourcecraft.dev/" class="tm-button code-explainer__link" style="visibility: hidden;"><img style="width:14px;height:14px;object-fit:cover;object-position:left;"/></a></div></pre><h3>Шаг 2. wget -O ./models/Qwen3.5-9B_Q4_k_m.gguf \
  "https://huggingface.co/SandLogicTechnologies/Qwen3.5-9B-GGUF/resolve/main/Qwen3.5-9B_Q4_k_m.gguf"</code><div class="code-explainer"><a href="https://sourcecraft.dev/" class="tm-button code-explainer__link" style="visibility: hidden;"><img style="width:14px;height:14px;object-fit:cover;object-position:left;"/></a></div></pre><h2>Запуск llama-server (REST API)</h2><p>После того как модель скачана, запускаем HTTP-сервер для взаимодействия через API:</p><pre><code class="bash">./bin/llama-server \
  -m ./models/Qwen3.5-9B_Q4_k_m.gguf \
  --host 0.0.0.0 \
  --port 8089 \
  -c 2048 \
  -ngl 99 \
  --jinja</code><div class="code-explainer"><a href="https://sourcecraft.dev/" class="tm-button code-explainer__link" style="visibility: hidden;"><img style="width:14px;height:14px;object-fit:cover;object-position:left;"/></a></div></pre><h3>Разбор ключевых параметров</h3><div><div class="table"><table><tbody><tr><td data-colwidth="262" width="262"><p align="left"><strong>Параметр</strong></p></td><td><p align="left"><strong>Описание</strong></p></td></tr><tr><td data-colwidth="262" width="262"><p align="left"><code>-c (--ctx-size)</code></p></td><td><p align="left">Размер контекста — длина "памяти" модели в токенах. Чем больше значение, тем длиннее диалог или документ она способна обработать. Однако это увеличивает потребление памяти и замедляет инференс. Для большинства задач хватает 4096–8192.</p></td></tr><tr><td data-colwidth="262" width="262"><p align="left"><code>-ngl (--n-gpu-layers)</code></p></td><td><p align="left">Количество слоев нейросети, загружаемых в видеопамять (GPU). Остальные слои будут работать на CPU. Чтобы достичь максимальной скорости, установите значение равным общему числу слоев модели (или укажите 999). Для слабых GPU можно снизить число, чтобы часть вычислений ушла на CPU.</p></td></tr><tr><td data-colwidth="262" width="262"><p align="left"><code>--jinja</code></p></td><td><p align="left">Включает поддержку шаблонов Jinja для правильной разметки диалогов (токены system/user/assistant). Без этого флага модель может "не понять" структуру чата и отвечать некорректно.</p></td></tr></tbody></table></div></div><h4>Дополнительные параметры для тонкой настройки</h4><div><div class="table"><table><tbody><tr><td data-colwidth="246" width="246"><p align="left"><strong>Параметр</strong></p></td><td><p align="left"><strong>Описание</strong></p></td></tr><tr><td data-colwidth="246" width="246"><p align="left"><code>-t (--threads)</code></p></td><td><p align="left">Количество потоков CPU для вычислений. По умолчанию — все доступные ядра. Можно ограничить, чтобы не нагружать систему.</p></td></tr><tr><td data-colwidth="246" width="246"><p align="left"><code>-tb (--threads-batch)</code></p></td><td><p align="left">Количество потоков для обработки батчей (промптов). Обычно меньше основного числа потоков.</p></td></tr><tr><td data-colwidth="246" width="246"><p align="left"><code>-b (--batch-size)</code></p></td><td><p align="left">Размер батча для обработки промптов. Влияет на скорость обработки входящего запроса.</p></td></tr><tr><td data-colwidth="246" width="246"><p align="left"><code>-ub (--ubatch-size)</code></p></td><td><p align="left">Размер микро-батча. Меньший размер экономит память, но может снизить скорость.</p></td></tr><tr><td data-colwidth="246" width="246"><p align="left"><code>--memory-f32</code></p></td><td><p align="left">Хранить кеш контекста в формате FP32 (точнее, но требует больше памяти).
