# Llama.cpp — Запуск и оптимизация локальной LLM

## Что такое Llama.cpp?

**Llama.cpp (LLaMA C++)** — это низкоуровневый фреймворк для выполнения эффективного вывода больших языковых моделей (LLM) на чистом C/C++. Проект был создан Георги Гержановым (@ggerganov), болгарским инженером-программистом, который разработал его вскоре после выпуска Meta LLaMA моделей, чтобы пользователи могли запускать их на обычном потребительском оборудовании без необходимости в дорогих GPU или облачной инфраструктуре.

Это один из самых влиятельных open-source AI проектов на GitHub с активной сообществом и постоянными обновлениями.

### Поддерживаемые модели

Llama.cpp поддерживает широкий спектр архитектур LLM:
- LLaMA 1, 2 и 3
- Mistral
- Phi
- Gemma от Google
- Yi
- Qwen
- Solar 10.7B
- Alpaca
- StableLM
- Falcon и RefinedWeb

## Основные возможности

### Чистый C/C++ без зависимостей
Полностью написан на чистом C/C++ без внешних зависимостей. Это означает, что Llama.cpp не требует Python runtime или сложных цепочек зависимостей, которые могут вызывать конфликты версий со временем. Весь код объединяется в один бинарный файл, который можно запустить практически где угодно — от высокопроизводительных серверов до Raspberry Pi.

### Ускорение оборудования
Llama.cpp поддерживает ускорение оборудования на всех основных платформах:
- **Apple M1/M2/M3/M4** — использует Metal для GPU вычислений с унифицированной архитектурой
- **AMD и Intel CPU** — оптимизированные AVX, AVX2 и AVX512 SIMD инструкции
- **NVIDIA GPUs** — CUDA с поддержкой вычислений с тензорными ядрами
- **AMD GPUs** — ROCm с помощью оптимизированных ядер

### Поддержка квантования
Llama.cpp включает отличные возможности квантования с разными уровнями точности от 2-бит до 8-бит. Система k-quants (Q4_K_M, Q5_K_S, Q6_K и т.д.) incorporates блочное квантование, которое также помогает сохранять качество модели при значительном уменьшении размера памяти. Например, модель на 7B параметров, которая обычно требует 14 ГБ для работы, может работать всего с 4 ГБ при 4-битном квантовании.

### Совместимый API OpenAI
Он поставляется со встроенным HTTP-сервером, который реализует спецификации API OpenAI. Это делает Llama.cpp достойной заменой вызовам API OpenAI. Он поддерживает следующие endpoints: /v1/completions, /v1/chat/completions, /v1/embedding в том же формате запроса и ответа.

### Множественные интерфейсы
Наслаждайтесь доступом через множественные интерфейсы, чтобы адаптировать различные типы рабочих нагрузок. CLI интерфейс предоставляет прямое взаимодействие с моделью LLM с полным контролем над параметрами. Интерактивный чат-режим предлагает разговорный опыт с постоянным контекстом и многооборотными диалогами. Встроенный HTTP/Rest сервер позволяет интеграцию с любым языком программирования или инструментом.

### Приватность
Вы имеете полный контроль над суверенитетом данных, так как это дает локальное выполнение. Все токены обработки остаются на вашем оборудовании, где вы его запускаете — никакие данные не отправляются ни на какие внешние серверы. Это может помочь пользователям, которые более ориентированы на приватность и безопасность и хотят обрабатывать конфиденциальные бизнес-документы, личную информацию или даже медицинские записи вместе с юридическими документами.

### Экономия памяти
Llama.cpp использует продвинутые техники оптимизации памяти, которые позволяют запускать большие модели на старом оборудовании с более низкими спецификациями. Memory mapping загружает модели напрямую с диска без необходимости копировать их в RAM, что уменьшает требования к памяти размером модели. KV cache квантование применяет 8-бит или更低 precision к key-value cache, сокращая использование памяти до 50% в среднем во время генерации.

### Продвинутые методы выборки
Он имеет сложные контролы генерации текста, которые позволяют вам тонко настраивать стиль вывода и качество. Temperature контролирует случайность — 0.1 для фокусированного и 1.0 для креативного. Top-p nucleus sampling динамически регулирует пул токенов на основе массы вероятности. Top-k ограничивает выбор k наиболее вероятными токенами. Repeat penalty предотвращает повторяющийся текст, наказывая недавно использованные токены.

## Архитектура и технологии

### ggml Tensor Library
Это основной вычислительный движок, построенный на ggml — пользовательской tensor библиотеке, написанной на C, которая предоставляет эффективные операции для ML inference с минимальными зависимостями.

### SIMD Оптимизация
Это ручная настройка ассемблера с использованием AVX, AVX2, AVX512 и NEON instruction sets для максимальной пропускной способности CPU при матричных операциях и других механизмах внимания.

### GPU Compute APIs
Llama.cpp имеет нативную интеграцию с CUDA, ROCm от AMD, Vulkan, Opencl и SYCL для ускоренного inference.

### KV Cache Management
Сложная key-value cache с поддержкой квантования, которая позволяет эффективное использование памяти во время длинных контекстных генерационных прогонов и разговоров.

### Build Systems
Llama.cpp поддерживает CMake, Make и различные платформо-специфичные инструменты сборки. У вас есть легкая компиляция across Linux, macOS, Android и Windows 10/11.

## Системные требования

### Минимальные требования
- C++11 compatible compiler
- 4GB RAM (для маленьких LLM моделей)
- Любой современный CPU
- Linux, macOS, или Windows

### Рекомендуемая конфигурация
- 16GB+ RAM
- Современный CPU с AVX2
- NVIDIA/AMD GPU (опционально)
- SSD для хранения моделей

### Поддерживаемые платформы
- Linux (x86, ARM)
- macOS (Intel & Apple Silicon)
- Windows (x86)
- Android, iOS, FreeBSD

### Опции ускорения
- NVIDIA CUDA (compute 6.0+)
- AMD ROCm
- Apple Metal
- Vulkan, OpenCL, SYCL

## Сравнение с другими инструментами

| Инструмент | Для чего лучше всего подходит | Примечания |
|------------|------------------------------|------------|
| **llama.cpp** | Настройка производительности, полный контроль над флагами, любое оборудование | Сборка из исходников; ориентирован на работу из командной строки (CLI) |
| **Ollama** | Нулевая конфигурация, управление моделями, Docker | Внутри использует llama.cpp; ограниченные возможности тонкой настройки |
| **LM Studio** | Настольный GUI, Windows/macOS, просмотр моделей, локальный API, совместимый с OpenAI | Хороший пользовательский интерфейс; поддерживает серверные/безголовые сценарии, JIT-загрузку, TTL и автоматическую выгрузку моделей |
| **vLLM** | Продакшн-сервинг для многих пользователей, непрерывная пакетная обработка (continuous batching) | Разработан для работы при полном заполнении VRAM; не подходит для гибридных конфигураций на потребительском оборудовании |
| **exllamav2** | Максимальная скорость для плотных моделей | Только CUDA; отлично подходит для моделей, которые полностью помещаются в VRAM |
| **mlx** | Apple Silicon | Только macOS; использует унифицированную память |
